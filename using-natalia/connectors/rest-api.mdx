## Overview

The REST API connector allows you to integrate external services with Natalia. Everything you configure here becomes part of Natalia's training sheet (main prompt), allowing her to naturally understand when and how to use these integrations.

### Important Notes
- All connector configurations are automatically added to Natalia's training
- Write clear descriptions - Natalia understands natural language
- Link connectors to scenarios where they'll be used

## Configuration in Dashboard

### Request Setup
1. **Name and Description**
- Give your connector a clear name
- Provide a detailed description of its purpose
- Example: "This connector retrieves order status from our system using the order reference number"

2. **API Configuration**
- Select the HTTP method (GET, POST, PUT, PATCH)
- Enter the API endpoint URL
- Configure headers and parameters as needed

3. **Parameters Description**

## Pro Prompting

For those who prefer structured data, here's how the configuration looks in YAML format:

```yaml
api_trigger:
  type: "order_lookup"
  conditions:
    - order_status_requested
    - tracking_requested
    - order_details_needed
  parameters:
    order_number:
      type: "string"
      description: "The order reference number"
    
query_conditions:
  type: "customer_info"
  parameters:
    email:
      type: "string"
      description: "Customer's email address"
```

## Best Practices

1. **Clear Descriptions**
- Use natural language in descriptions
- Be specific about when to use the connector
- Describe expected outcomes

2. **Testing**
- Test with various inputs
- Verify error handling
- Validate responses

3. **Security**
- Use environment variables for sensitive data
- Implement proper authentication
- Follow least privilege principle

## Remember
- All connector settings are automatically added to Natalia's training
- No need to manually code the integrations
- Link connectors to relevant scenarios
- Test thoroughly before going live
