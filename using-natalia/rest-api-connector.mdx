## Overview

The REST API connector allows you to integrate external services and APIs with Natalia. This connector enables you to send HTTP requests to any endpoint and use the responses in your Natalia scenarios.

## Configuration

### Basic Setup

1. Navigate to the Connectors section in your Natalia dashboard
2. Click "+" to create a new connector
3. Select "Custom API" as the connector type
4. Fill in the following information:
   - Connector name
   - Description of the connector's purpose
   - HTTP method (GET, POST, PUT, PATCH)
   - API endpoint URL

### Request Configuration

The REST API connector supports:

1. **HTTP Methods**
- GET: Retrieve data
- POST: Create data
- PUT: Update data
- PATCH: Partial updates

2. **Headers**
Add custom headers like authorization tokens or content types.

3. **Parameters**
Configure URL parameters and query strings.

4. **Body**
For POST/PUT/PATCH requests, define the request body structure:
- Support for string, number, object and array types
- Nested object structures
- Arrays of items

## Body Structure Example

```json
{
  "user": {
    "type": "object",
    "required": [
      "name",
      "email"
    ],
    "properties": {
      "name": {
        "type": "string",
        "description": "User's name"
      },
      "email": {
        "type": "string",
        "description": "User's email address"
      }
    }
  },
  "description": {
    "type": "string",
    "description": "Request description"
  }
}
```

## Best Practices

1. **Error Handling**
- Always include error responses in your API documentation
- Test your connector with invalid inputs
- Handle rate limits appropriately

2. **Security**
- Use environment variables for sensitive data
- Implement proper authentication
- Follow the principle of least privilege

3. **Documentation**
- Document the purpose of each field
- Include example requests and responses
- Note any dependencies or prerequisites

4. **Testing**
- Test with various data types
- Verify error scenarios
- Validate response handling

## Using in Scenarios

Once configured, you can use the REST API connector in your Natalia scenarios:

1. Reference the connector by its name
2. Pass required parameters
3. Handle responses appropriately
4. Implement error checking

Example usage in a scenario:
```yaml
- name: "Create Ticket"
  connector: "ticket-creator"
  params:
    user:
      name: "John Doe"
      email: "john@example.com"
    description: "Need assistance with login"
```

## Troubleshooting

Common issues and solutions:

1. **Connection Errors**
- Verify endpoint URL is correct
- Check network connectivity
- Validate authentication credentials

2. **Invalid Responses**
- Review request format
- Check API documentation
- Validate data types

3. **Performance Issues**
- Monitor response times
- Implement caching where appropriate
- Consider rate limiting
